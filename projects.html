Occam's Vault

I use many self-hosted services but have not developed one myself. I decided to create a network attached vault that stores credentials, notes and files for multiple users while providing robust storage.

This project started as a means to explain self-hosting to my friends and family and providing them with a local cloud.

  • Key Features:
    • Data Security: Utilizes AES encryption for data at rest and Argon2ib for hashing the master password. No credentials are stored in plaintext, ensuring the highest level of security.
    • Privacy & Availability: The application is designed to ensure privacy, with no data transferred outside the network.
    • User Control: Users can export their database to other password managers by downloading their cached database, ensuring flexibility and full control over their data. Users can also completely delete their user profile.
  • Technology Stack:
    • Backend Development: Python and the Bottle framework
    • Database: MySQL for secure and reliable data storage
    • Network Security: SSL/TLS using OpenSSL for secure communication over HTTPS

Project Impact: Occam's Vault provides a seamless experience for users seeking a self-hosted password management solution. Its design ensures data integrity, availability, and confidentiality, making it a comprehensive tool for managing sensitive information securely.
